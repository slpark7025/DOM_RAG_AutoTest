Move Field

설명

어떤 필드가 자신이 속한 모듈보다 다른 모듈에서 더 많이 사용될 때는 대상 모듈 안에
새 필드를 선언하고 그 필드 참조 부분을 전부 새 필드 참조로 수정한다.


방법


1. 필드가 public이면 필드 캡슐화 기법을 실시한다.
2. 컴파일과 테스트를 실시한다.
3. 대상 모듈 안에 읽기/쓰기 메서드와 함께 필드를 작성한다.
4. 대상 모듈을 컴파일한다.
5. 원본 필드나 함수에서 대상 모듈을 참조할 방법을 정한다.
6. 원본 모듈에서 필드를 삭제한다.
7. 원본 필드를 참조하는 모든 부분을 대상 모듈에 있는 적절한 함수를 참조하게 수정한다.

Move Method

설명

함수가 자신이 속한 모듈보다 다른 모듈의 기능을 더 많이 이용할 땐
그 함수를 제일 많이 이용하는 모듈 안에서 비슷한 내용의 새 함수를 작성한다.
기존 함수는 간단한 대리 함수로 전환하든지 아예 삭제한다.


방법


1. 원본 모듈에 정의되어 있는 원본 함수에 사용된 모든 기능을 검사한다.기능을 전부 옮겨야 할지 판단한다.
2. 원본 모듈의 하위모듈과 상위모듈에서 그 함수에 대한 다른 선언이 있는지 검사한다.
3. 그 함수를 대상 모듈 안에 선언한다.
4. 원본 함수의 코드를 대상 함수에 복사한 후, 대상 모듈 안에서 잘 돌아가게끔 대상 함수를 약간 수정한다.
5. 대상 모듈을 컴파일 한다.
6. 원본 객체에서 대상 객체를 참조할 방법을 정한다.
7. 원본 함수를 위임 함수로 전환한다.
8. 컴파일과 테스트를 실시한다.
9. 원본 함수를 삭제하든지,아니면 위임 함수로 사용하게 한다.
10. 원본 함수를 삭제할 때는 기존의 참조를 전부 대상 함수 참조로 수정한다.
11. 컴파일과 테스트를 실시한다.

Decompose conditional

설명

복잡한 조건문(if-then-else)이 있을 땐 if, then, else 부분을 각각 메서드로 빼낸다.


방법

1. if절을 별도의 메서드로 빼낸다.
2. then절과 else절을 각각의 메서드로 빼낸다.

Before After
if(data_befor(SUMMER_START) || data_after(SUMMER_END)){
   charge = quantity * _winterRate + _winterServiceCharge;
} else {
   charge = quantity * _summerRate;
}
 if(notSummer(date)) {
    charge = winterCharge(quantity);
 } else {
    charge = summerCharge(quantity);
 }
Remove Parameter

설명

함수가 어떤 매개변수를 더 이상 사용하지 않을 땐 그 매개변수를 삭제한다.


방법


1. 함수 시그니처가 다른 모듈에서 참조되어 있는지 검사한다.
2. 다른 모듈에서 제거하려는 매개변수를 사용하는지 검사 후 사용한다면 리팩토링 하지 않는다.
3. 제거할 매개변수가 없는 새 함수를 선언하고,원본 함수의 내용을 새 함수로 복사한다.
4. 컴파일 한다.
5. 새 함수가 호출하도록 원본 함수의 내용을 수정한다.
6. 컴파일과 테스트를 실시한다.
7. 원본 함수 참조 부분을 전부 찾아서 새 함수를 참조하도록 수정한다.
8. 수정할 때마다 테스트를 실시한다.
9. 원본 함수를 삭제한다.
10. 컴파일과 테스트를 실시한다.

Pull Up Method

설명

기능이 같은 함수가 여러 하위모듈에 들어있을 땐 그 함수를 상위모듈로 옮긴다.


방법


1. 함수가 서로 같은지 검사한다.
2. 함수의 시그니처가 서로 다르다면 모든 시그니처를 상위모듈에 사용하고자 하는 시그니처로 수정한다.(복잡도 고려)
3. 상위모듈 안에 새 함수를 작성하고 새 함수 안에 같은 함수의 내용을 복사한 후 적절히 수정하고 컴파일한다.
4. 하위모듈의 함수를 하나 삭제한다.
5. 컴파일과 테스트를 실시한다.
6. 상위모듈의 함수만 남을 때까지 하위모듈의 함수를 계속 삭제한다.
7. 상위모듈 함수를 호출하는 부분을 보면서 필요한 타입을 상위모듈로 수정할 수 있는지 파악한다.

Push Down Method

설명

상위모듈에 있는 기능을 일부 하위모듈에서만 사용할 땐 그 기능을 관련된 하위모듈 안으로 옮긴다.


방법


1. 하위모듈에 함수를 하나 선언하고 그 함수의 내용을 각 하위모듈로 복사한다.
2. 상위모듈의 함수를 삭제한다.
3. 컴파일과 테스트를 실시한다.
4. 그 함수가 필요 없어진 하위모듈에서 그 함수를 삭제한다.
5. 컴파일과 테스트를 실시한다.

Replace Method with Method Object

설명

지역변수 때문에 함수 추출을 적용할 수 없는 긴 함수가 있을 땐
그 함수 자체를 새로운 모듈로 전환해서 모든 지역변수를 새 모듈의 필드로 만든다.
그런 다음 그 함수를 모듈 안의 여러 메서드로 분리한다.


방법

1. 전환할 함수의 이름과 같은 이름으로 새 모듈을 생성한다.
2. 새 모듈에 원본 함수가 들어 있던 모듈의 헤더파일을 추가하고 원본 함수 안의 각 임시변수와 매개변수에 해당하는 속성을 추가한다.
3. 새 모듈에 각 매개변수를 받는 초기화 함수를 작성한다.
4. 새 모듈에 compute라는 이름의 함수를 작성한다.
